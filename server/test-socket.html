🔒 Security Architecture Explained:
Each user gets a unique Socket ID (this is correct!):
• Teacher in Tab 1: Socket ID "Ag_glTZWu0E4elLUAAAF"

Session Registration links Socket ID to Role:
• When you register, server stores: {socketId → role → sessionId → name}
• Server validates EVERY action by checking the socket's registered session
• Client can't fake sessionId or name - server uses validated session data!
